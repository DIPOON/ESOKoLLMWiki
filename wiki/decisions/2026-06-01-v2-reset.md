# v1 → v2 Reset: ESO-first 학술 wiki로 재편

## 배경

v1 (`feature/uesp_ingest` 브랜치의 2026-05-29 ~ 06-01 세션)에서 Karpathy LLM Wiki 패턴을 ES 도메인에 적용해 1616 페이지·989 raw processed·360 commit까지 양적 cascade를 수행. 그러나 사용자가 `/mnt/e/ESOKoLLMWikiByCodex/`의 Codex 구현 (560 페이지·129 raw·19 commit)과 비교 요청한 결과, 두 위키의 *질적 차이*가 분명해짐.

## v1의 강점 (보존할 가치)

- 양적 규모와 ES 전체 도메인 coverage
- Orphan 0 정책 일관 유지
- 빠른 cascade를 통한 광범위 navigation 가능성
- 각 batch당 1 commit으로 작업 history 추적 가능

## v1의 결정적 약점 (v2에서 교정)

1. **출처 추적 불가**: 페이지 claim이 raw 어디서 왔는지 추적 시스템 없음
2. **내적 ES 지식 의존**: 다수 stub이 raw 미독해 + 일반 ES 지식으로 작성 → 검증 불가능한 claim 다수
3. **시대 혼합**: ESO 2E 582 페이지에 3E·4E 정보가 mark 없이 섞임
4. **페이지 깊이 얕음**: 평균 40줄 (Codex 130줄)
5. **stub 남발**: 50+ batch에서 빠른 양산 → 다수가 4-6줄 stub
6. **frontmatter 불일치**: `term:` vs `name:` 혼재, aliases 인용부호 오류
7. **`*` asterisk 본문 강조 남발**: 가독성 해침
8. **layer separation 부재**: raw 노드가 Obsidian vault graph에 노출, raw 내부 UESP MediaWiki 마크업이 vault scope에서 해석되어 graph 오염
9. **commit 낭비**: 360 commit이 source가 아닌 *배치* 단위 → provenance 불명확

## v2 결정 사항

### 보존

- raw/ immutable 원칙
- wiki/decisions/ 메타-결정 폴더
- Orphan 0 정책 (단 *결과*로서 — 강제 link 추가 금지)
- 번역 plugin 후순위 미션

### 변경

| 항목 | v1 | v2 |
|---|---|---|
| 폴더 구조 | flat (wiki/*.md) | 분류 폴더 (people/places/concepts/factions/events/quests/texts/source-notes/decisions) |
| Page naming | kebab-case 한글 가능 | **영문 canonical** (예: `Molag Bal.md`) |
| 본문 언어 | 한국어 + `*asterisk*` 남발 | 한국어 + 표준 markdown |
| 도메인 범위 | ES 전체 (Aedric/Daedric/4 Era 평등) | **ESO-first** (2E 582 frame), 후대는 Broader TES Context로 분리 |
| Source 추적 | 페이지 하단 raw 경로 한 줄 | **wiki/source-notes/ 별도 시스템** (Key Claims Extracted + Source Limits + Follow-Up Sources) |
| Stub 생성 조건 | 내적 ES 지식으로 자유 | **raw에 substantive 언급 있을 때만** |
| Commit 단위 | 배치 (1 batch = 1 commit) | **1 source = 1 commit** |
| frontmatter | `term:`, `kind:`, `category:` | `type:`, `name:`, `aliases:`, `era:`, `appears_in:`, `related:`, `status:`, `source_priority:` |
| Page sections | 자유 | **Summary / ESO Context / Broader TES Context / Chronology / Related / Interpretive Issues / Sources** 표준 |
| Layer separation | wiki ↔ raw 경계 모호 | **raw/는 Obsidian vault scope에서 제외**, wiki [[X]]는 wiki/.../X.md 보장 |

### Agent Roles 신규

- **Claude (Anthropic)**: primary maintainer. 본 위키의 주된 ingest·페이지 작성·유지보수
