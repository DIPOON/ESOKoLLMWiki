# Style Guide — ESO 한국어 번역 가이드

이 문서는 번역 결정의 *룰 모음*이다. CLAUDE.md §4 워크플로 step 1에서 *반드시* 읽고 번역에 적용한다.

**출처**: 한국 ESO 커뮤니티 통일안 *"번역자들 참고논의"* + *"npc말투"* 시트에서 정수 추출.
**원본 보존**: [[decisions/seed-policy]] + [[decisions/seed-npc-tone]].

---

## 1. 명명 형식 — 텍스트 유형별

### 1.1 퀘스트명
- **형식**: `번역(원문)`
- 예: `거인의 딸 (Daughter of Giants)`

### 1.2 인명
- **발음을 알면**: `음차`
  - 예: `리리스 타이탄본`
- **발음이 부정확하거나 캐릭터 발음이 안 나오면**: `음차(원문)`
  - 예: `리리스 타이탄본(Lyris Titanborn)`
- **아르고니안 이름**: 음차 (오블리비언/스카이림 관례).
  - 복수형 `s`가 어색하면 *제거*.
  - 예시:
    - `Lifts-her-tail` → `리프트-허-테일` (❌ 리프츠-허-테일)
    - `Sees-All-Colors` → `씨-올-컬러스` (❌ 씨즈-올-컬러스. 인게임 발음 "씨즈-올-칼라즈"는 별개)
    - `Reads-the-Winds` → `리드-더-윈드`
    - `Born-From-Pits` → `본-프롬-핏`

### 1.3 지명
- **지역·도시 (큰 곳)**: `음차 (원문)` 또는 `음차`
  - 예: `앤빌 (Anvil)`
- **도시 내 건물 (여관·상점 등)**: `번역`
  - 예: `Silver Moons Inn` → `은빛 달 여관`

### 1.4 일반 아이템
- **원문 필요** (거래·위키검색 용도): `번역 (원문)`
  - 예: `조리법: 아르고니안식 훈제 블랙마쉬 튀김 (Recipe: Argonian Smoked Blackmarsh Fries)`
- **원문 불필요** (도둑질 아이템 등): `번역`만
  - 예: `늑대인간의 머스크`

### 1.5 재료명
- **형식**: `번역 (원문)`
- 예: `루비다이트 주괴 (Rubedite Ingot)`

### 1.6 스킬명 ⚠
- **형식**: **원문 그대로 (번역 안 함)**
- 예: `Wall of Fire` → `Wall of Fire`
- 이유: 애드온 호환, 자료 검색, 갤러리 검색.

---

## 2. 세트 아이템 명명

### 2.1 접미형 (예: `Jack of the Lich`)
- **형식**: `The <Set>: <아이템 한국어>`
- 예: `Jack of the Lich` → `The Lich: 리치의 경갑`
- ⚠ `the`는 *반드시 `The`로 대문자*. 안 그러면 아이템 링크 오류.
- 룰 유래: 한패 이름만으로 영어 아이템 검색 가능하도록 (`'s`를 기준으로 나눔).

### 2.2 접두형 (예: `Meritorious Gloves`)
- **형식**: `<Set 영문>: <아이템 한국어>`
- 예: `Meritorious Gloves` → `Meritorious: 마땅한 칭찬의 장갑`
- 던전세트(Guise / Mask 등) 표현은 [[termbase]] 아이템 카테고리 참조.

### 2.3 유니크 (네임드 단일 아이템)
- **형식**: `<Set명>: <한국어>`
- 예: `Lost Imperial Band` → `Netch's Touch: 유실된 임페리얼 반지`
- 통일성: 세트명을 *앞에* 붙임.
- 예외: 영문 네임드 이름이 *그대로 나은 경우*도 있음. 본인 판단.

## 3. NPC 말투

### 3.1 일반 원칙
- *NPC → 플레이어* / *플레이어 → NPC* 말투를 *개별 관계*마다 결정.
- 캐릭터 성격 / 사회적 지위 / 친밀도 반영.
- **인접 ±3 인덱스 내 톤 일관성 유지** (같은 다이얼로그 묶음의 화자가 갑자기 톤 바뀌면 안 됨).
- 새 NPC 만나면 §3.3 표 또는 [[lore]] 페이지에 톤 기록 (**fix-as-you-go**, CLAUDE.md §4 step 1).

### 3.2 카짓(Khajiit) 특수 규칙
- **기본**: *3인칭화*.
- `This one` → `이 몸`
- `This khajiit` → `이 카짓`
- 예외: `I` → `나` (특수한 경우만, 일인칭)

### 3.3 알려진 NPC 톤

| NPC | NPC → 플레이어 | 플레이어 → NPC | 비고 |
|---|---|---|---|
| 바누스 갈레리온 (Vanus Galerion) | "~하게" 체 | 존댓말 | |
| 네라모 (Neramo) | 반말 | 반말 | 드웨머 유적 / 클락워크시티 DLC. 친근. 플레이어를 "조수"라 부름. |
| 아말리엔 (Amalien) | 반말 | — | 유물 코덱스, 알트머 고고학자 |
| 가브리엘 베넬레 (Gabrielle Benele) | 존댓말 | — | 대거폴 / 콜드하버 / 고고학 코덱스. 스토리 대화와 유물 설명 모두 존칭. |
| 우그론 그로-투모그 (Ugron gro-Thumog) | 반말 | — | 유물 코덱스. 오시머 고고학자. 그와일림 대학 무덤학 교수. |
| 레지누스 부카 (Reginus Buca) | "~라네" / "~하게" 체 | — | 유물 코덱스. 브레튼 고고학자. 그와일림 대학 교수. 연장자처럼. |
| 베리타 누미다 (Verita Numida) | "~요" 경어 | — | 유물 코덱스. 브레튼. 사무적 (플레이어 높이는 게 아니라 사무적). |

> 시드의 표본은 *유물 코덱스 NPC* 위주. 일반 NPC 톤은 사례마다 결정 + §3.3 표에 추가.

## 4. 퀘스트 번역 룰

- 괄호 / 주석을 *제거*. 필요한 정보면 지문에 풀어 적기.
- 기존에 들어있는 괄호 / 주석도 *제거*.
- 셰오고라스같이 광기 캐릭터는 격식체보다 *광기체*가 자연스러움 (사례별 [[decisions]] 기록).

---

## 5. 가구 명칭

- **미정**. 시리즈(Basin / Gilded / Rustic / Verdant / Overhanging 등) 통일 필요.
- 가구 설명문(예: "This is a standard house item")은 *번역 대상*.

---

## 6. 특수 문자·시스템 토큰 ⚠ (절대 보존)

게임 텍스트에 다음 패턴은 *시스템이 해석*하는 토큰. **번역 중 지우거나 변경 금지.** 의심스러우면 보존 후 검토.

### 6.1 변수
- 형식: `<<숫자>>` (예: `<<1>>`, `<<2>>`)
- 의미: 게임 런타임에 변수값(골드 수·아이템 수 등) 삽입
- 예: `You have <<1>> golds` → `당신은 <<1>> 골드를 가지고 있습니다`

### 6.2 색상
- 형식: `|cffffff내용|r` (cffffff는 #ffffff 헥스 색상)
- 의미: 텍스트 색상 지정
- 예: `|cff0000경고|r` → `|cff0000경고|r` (내부 텍스트만 번역)

### 6.3 단수/복수
- 형식: `<<숫자[$d 단수형/$d 복수형]>>`
- 의미: 변수값에 따라 단/복수 선택
- 예: `<<1[$d second/$d seconds]>>` → 값이 1이면 "second", 그 외면 "seconds"
- 한국어는 단복수 구분 약함 — *통합 표현*(예: "$d초")이 자연스러울 수 있음. 사례별 판단.

### 6.4 ID 문자열
- 형식: `Q\d+`, 영문 ID 문자열 (예: `Q4355 Mirkalinde's House`)
- 처리: *그대로 보존*. 번역 X.
- 예: `Q4355 Mirkalinde's House` → `Q4355 Mirkalinde's House`

### 6.5 명사 속성 (성별·위치)
- `^M` 남성 / `^F` 여성 / `^N` 중성 (고유 명사)
- `^m ^f ^n` 보통 명사 성별
- `^a ^z` 어순 변경 (수식어 위치)
- 처리: *그대로 보존*. 한국어는 명사 성별 없지만 토큰은 유지.
- 예: `리리스 타이탄본^F`

## 7. 검수·맞춤법

- 번역 완료 후 *한국어 맞춤법 검사기* 돌리기 권장.
- 긴 문장은 *문장 단위로 잘라* 번역 OK.
- 검수 미통과 표시 (관례):
  - 기계 번역(구글): 끝에 `-G-`
  - 퍼지: 별도 표시

---

## 8. 자주 헷갈리는 케이스

- **`Adept`**: 의미 다양 (숙련생 / 숙련자 / 명인 / 신봉자). 컨텍스트 보고 결정. → [[adept]]
- **`Mage`**: 메이지 / 마법사 / 마도사 혼용. 컨텍스트 보고 결정. → [[mage]]

---

## 9. 본 가이드 갱신 정책

- 번역 중 *기존 룰로 해결 안 되는* 케이스 발견 → `wiki/decisions/YYYY-MM-DD.md`에 기록 + 본 가이드 룰 추가.
- 새 NPC 톤 결정 → §3.3 표에 행 추가.
- 동일 종류 마찰 *2회 이상* → 본 가이드의 *정식 룰*로 승격.
- 본 가이드는 *살아있는 문서*다. CLAUDE.md §12 "점진적 개선" 정신과 동일.
